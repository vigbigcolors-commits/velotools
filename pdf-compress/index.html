How it works

How our browser PDF compressor works

Unlike most online tools, we don't send your files to a server. Here's exactly what happens inside your browser when you compress a PDF.

01

PDF is loaded into browser memory

The PDF file is read using the FileReader API — entirely locally. The raw bytes never touch a network connection. The PDF.js engine parses the document structure, detecting all pages, fonts, and images.

02

Each page is rendered to a canvas

Every page is drawn onto an HTML Canvas element at your chosen DPI. Lower DPI means a smaller canvas, which directly reduces the output file size. This is the main lever for compression.

03

Canvas is encoded as JPEG

The rendered canvas is converted to a JPEG image at your chosen quality percentage (10–95%). JPEG compression is lossy but extremely efficient — this is where most of the size reduction comes from.

04

New PDF is assembled with pdf-lib

All compressed page images are assembled into a new, clean PDF using the pdf-lib library. Metadata, annotations, and embedded fonts from the original are removed or retained based on your settings.

Tips

Get the most out of PDF compression

📷

Scanned documents compress best

Scanned PDFs are essentially images inside a PDF wrapper — they compress 70–90% with the Screen or Web preset. This is the primary use case for PDF compression.

📋

Use grayscale for black & white docs

If your document doesn't need colour (contracts, letters, reports), enable Grayscale mode. It removes all colour data and adds an extra 10–20% compression on top of the JPEG reduction.

💾

Text PDFs compress less

Native text-based PDFs (created in Word, InDesign, etc.) don't have large image data to compress. You'll still see some reduction from metadata removal, but expect 10–30% rather than 80%+.

🔌

ZIP download for multiple files

When compressing a batch of files, use "Download All (ZIP)" — it packages all compressed PDFs into a single archive. No need to download and rename each file manually.

FAQ

Frequently asked questions about PDF compression

Does this PDF compressor upload my files to a server?
Absolutely not. Every step of the compression process happens in your browser using JavaScript. Your PDF file never leaves your device and is never sent to any server, anywhere. You can verify this by turning off your internet connection after the page loads — compression still works perfectly.
How much can I reduce a PDF file size?
It depends on the content. Image-heavy PDFs (scanned documents, photo books, presentations with photos) typically compress 50–90%. The Screen preset with a scanned document often achieves 85% or more. Text-only PDFs compress 10–25% mainly from metadata removal.
Will compression reduce the quality of my PDF?
Yes — PDF compression involves a quality trade-off. The Web preset (65% JPEG quality, 96 DPI) is barely noticeable on screen. The Screen preset (40% JPEG, 72 DPI) shows some quality reduction at close zoom. Use the Archive preset if quality preservation is critical.
Is there a file size limit?
There's no hard limit — the only constraint is your device's available memory. Most PDFs up to 100MB process without issues. Very large files (200MB+) may be slow or cause memory warnings on older devices.
Can I compress multiple PDF files at once?
Yes. Drag and drop up to 20 PDF files at once, or select multiple files in the file picker. Each file shows its own progress bar and compression result. Download them individually or all together as a ZIP archive.
What does "Strip metadata" do?
PDFs contain hidden metadata: author name, creation date, modification date, application used to create the file, and sometimes GPS coordinates. Stripping metadata removes this personal information and slightly reduces file size.
Why does my text PDF barely compress?
Our compressor works by re-rendering pages as JPEG images. This is extremely effective for image-heavy PDFs but less so for text-only PDFs, because clean text renders as efficient vector data rather than image data. For text PDFs, the main savings come from metadata removal.
